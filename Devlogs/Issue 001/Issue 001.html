How I made Gold Rush

Hey fellas. It's been about 3 months since I've released Gold Rush. I figured I'd go into my thought process when I was developing the game to hopefully give some perspective about what making my first game was like.

The idea came to me sometime in early December. I had just gotten a certification at EVIT, and wanted to pick up and learn Godot. So I had an idea for an idle game. I figured it'd be a simple learning project.

But I had a problem: Idle games are boring.

And I didn't want to make something that was boring. I wanted to make something that was actually (at least a little bit) fun. Not something that you'd just leave running in the background when you play something else. So I thought about how to make an idle game more interesting. And ultimately, I landed on adding a time limit with a quota.

Within a couple days, I had a prototype put together. Got some feedback on the concept, and people seemed to like it. So I started properly putting together the game.

[DECLASSIFIED]

Gold Rush prototype screenshot from early Janurary.


When I made the game itself, it was actually a separate Godot project from the original prototype. When I was working on the prototype, the code was a mess. This was mainly because I was almost brand new to working in Godot, and just wanted to get a concept off the ground. So I made a whole new Godot project with the logic that I'll refactor all the code later (I did, in fact, NOT do that).

When I was actually making the final game, I didn't actually do that much refactoring of the code. And it got worse around the time of the game's release when I just wanted to be done with it. On the surface, the game seems fine, but the code itself was a huge pain in the ass to scale.

[DECLASSIFIED]

A screenshot of the Gold Rush "animation managers". Systems like this exist for every "screen" in the game (the game itself, the shop, etc.)


During the final week before the game dropped, I passed it around to people at EVIT and gave them one job: break the game as badly as they could.


One guy was able to completely break how the animations looked and played. Hell, one guy even figured out how to get negative gold. Yeah it was a pain in the ass to fix all those problems, but it did, objectively, make the game better for the release.

The day after the release, I got to have the game running on a computer at EVIT expo and some people played it. That felt super cool, and one guy even saved the link to play it again later.

I put out a v1.1 update and haven't touched the project since. And I can't say I plan on it right now. But I did really enjoy working on this first project, despite how messy the code was. It made me get familiar with actually finishing my projects. And it's pushed me to keep working on other stuff.

If you've played Gold Rush and enjoyed it, I really do appreciate it. And if you've read this far, I appreciate that you were willing to read my unedited yapping. Thanks bro.
